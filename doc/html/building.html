Building QuickFIX

Building with CMake (Recommended)

QuickFIX uses CMake as its primary build system, providing consistent builds across all platforms.

Prerequisites

  • C++17 compatible compiler (GCC 7+, Clang 5+, MSVC 2017+)
  • CMake 3.5 or higher
  • Optional: OpenSSL for SSL/TLS support
  • Optional: MySQL, PostgreSQL, or ODBC client libraries

Windows

Basic Build:

mkdir build
cd build
cmake -G "Visual Studio 17 2022" -A x64 ..
cmake --build . --config Release
cmake --install . --config Release

With SSL Support:

mkdir build
cd build
cmake -G "Visual Studio 17 2022" -A x64 ^
  -DHAVE_SSL=ON ^
  -DOPENSSL_ROOT_DIR="C:\path\to\openssl" ^
  -DCMAKE_INSTALL_PREFIX="C:\quickfix" ..
cmake --build . --config Release
cmake --install . --config Release

With Database Support:

cmake -G "Visual Studio 17 2022" -A x64 ^
  -DHAVE_MYSQL=ON ^
  -DHAVE_POSTGRESQL=ON ^
  -DHAVE_ODBC=ON ..
cmake --build . --config Release

Supported Visual Studio versions: VS 2017, VS 2019, VS 2022


Linux / macOS

Basic Build:

cmake -DCMAKE_BUILD_TYPE=Release .
make -j$(nproc)
sudo make install

With SSL Support:

cmake -DCMAKE_BUILD_TYPE=Release -DHAVE_SSL=ON .
make -j$(nproc)
sudo make install

With Database Support:

cmake -DCMAKE_BUILD_TYPE=Release \
  -DHAVE_MYSQL=ON \
  -DHAVE_POSTGRESQL=ON .
make -j$(nproc)
sudo make install

Custom Installation Directory:

cmake -DCMAKE_BUILD_TYPE=Release \
  -DCMAKE_INSTALL_PREFIX=/usr/local .
make -j$(nproc)
sudo make install

CMake Build Options

Option Description Default
-DHAVE_SSL=ON Enable SSL/TLS support (requires OpenSSL) OFF
-DHAVE_MYSQL=ON Enable MySQL message store support OFF
-DHAVE_POSTGRESQL=ON Enable PostgreSQL message store support OFF
-DHAVE_ODBC=ON Enable ODBC message store support OFF
-DHAVE_PYTHON3=ON Build Python 3 bindings OFF
-DQUICKFIX_SHARED_LIBS=ON Build shared libraries instead of static ON
-DQUICKFIX_EXAMPLES=ON Build example applications ON
-DQUICKFIX_TESTS=ON Build test suite ON
-DCMAKE_BUILD_TYPE=Release Build type (Debug, Release, RelWithDebInfo) -
-DCMAKE_INSTALL_PREFIX=/path Installation directory /usr/local

Building with Autotools (Alternative)

QuickFIX also supports traditional autotools-based builds on Unix-like systems.

Linux / Solaris / FreeBSD / macOS

./bootstrap
./configure
make
make check
sudo make install

You can run configure with the --prefix=<base directory> option to install to a custom location.

./configure options:

--prefix=/usr/local
Installation directory
--with-openssl[=PATH]
Enable SSL support, optionally specify OpenSSL installation path
--with-python3
Build the Python 3 bindings
--with-ruby
Build the Ruby bindings
--with-mysql
Compile with MySQL support
--with-postgresql
Compile with PostgreSQL support

Platform-Specific Notes

Solaris with SUNPro Compiler

See README.SunOS for detailed instructions. When using the SUNPro compiler:

CC=<full path to SUNPro CC binary>
CFLAGS=-g -xs
CXX=<full path to SUNPro CC binary>
CXXFLAGS=-g -xs
LD=<full path to SUNPro CC binary>
LDFLAGS=-g -xs -lCstd
AR=<full path to SUNPro CC binary>
AR_FLAGS=-g -xs -xar -o

AIX

See README.AIX for detailed build instructions on AIX systems.

FreeBSD

FreeBSD is supported via autotools. CMake support may require minor adjustments.


Testing Your Build

With CMake:

cmake --build . --target test
# or
ctest

With Autotools:

make check

Using vcpkg

QuickFIX is available via the vcpkg package manager:

git clone https://github.com/Microsoft/vcpkg.git
cd vcpkg
./bootstrap-vcpkg.sh  # Windows: bootstrap-vcpkg.bat
./vcpkg integrate install
./vcpkg install quickfix

Troubleshooting

  • Ensure your compiler supports C++17
  • For SSL issues, verify OpenSSL is properly installed and OPENSSL_ROOT_DIR is set correctly
  • For database support, ensure client libraries are installed and in the library path
  • On Windows, run CMake from the "Developer Command Prompt for Visual Studio"
  • Check the GitHub Issues page for known build problems
